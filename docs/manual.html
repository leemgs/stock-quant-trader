1. config.yaml 완벽 마스터

시스템의 두뇌인 설정 파일을 본인의 스타일에 맞게 튜닝하세요.

항목설명권장값
max_budget자동매매에 사용할 총 예산계좌 잔고의 50%
stop_loss손절매 기준 (비율)0.015 (-1.5%)
k_value돌파 민감도 (낮을수록 공격적)0.4 ~ 0.6
gemini_api_keyAI 뉴스 분석용 API 키필수

2. 프로그램 가동 프로세스

시스템을 실행하는 두 가지 방법이 있습니다.

[방법 1] 일반 Python 실행

  1. API 상태 점검: KIS API 연결 상태를 먼저 확인하세요.
  2. main.py 실행: python main.py를 실행합니다.
  3. 텔레그램 확인: "자동매매가 시작되었습니다" 메시지가 오면 정상입니다.

[방법 2] Docker Compose 백그라운드 실행 (권장)

Docker 환경에서는 아래 명령어 한 줄로 매매 봇과 실시간 대시보드가 동시에 가동됩니다.

docker-compose up -d
  • 봇 실시간 로그 확인: docker-compose logs -f bot
  • 대시보드 접속: 웹 브라우저에서 http://localhost:8501 접속
  • 시스템 안전 종료: docker-compose down

3. 실시간 대시보드 활용

streamlit run src/monitor/dashboard.py 명령어로 웹 대시보드를 켭니다.

  • Equity Curve: 자산의 우상향 여부를 체크하세요.
  • Heatmap: 오늘 어떤 종목이 효자 종목인지 시각적으로 확인하세요.
  • AI Review: 장 종료 후 AI가 분석한 오늘의 매매 복기를 정독하세요.

4. 자주 묻는 질문 (FAQ)

Q: 주문이 나가지 않아요.

A: 1. 예수금이 부족하지 않은지 확인하세요. 2. 모의투자 계좌인 경우 한국투자증권 홈페이지에서 '모의투자 신청'이 되어 있는지 확인하세요.

Q: Ubuntu 서버에서 백그라운드로 실행할 수 있나요?

A: 네, 가능합니다. nohup, tmux, docker 등을 활용하여 24시간 가동할 수 있습니다.
